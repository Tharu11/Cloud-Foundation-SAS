19_SAS_On Cloud Models Shared Responsibility Model.




1. How many types of cloud models are available?
ANS. There are four types of cloud models: On Premises, Infrastructure as a service, Platform as a service, and Software as a service.
	

2. With an on premises cloud model what does an organization/user manage and what is managed by a cloud service provider?
ANS. With an on premise cloud model an organization/user manages Applications, Data, Runtime, Middleware, O/S, Virtualization, Servers, Storage, Networking. Nothing is managed by cloud service providers because we donâ€™t have their services.
	

3. With infrastructure as a service cloud model what is managed by a cloud service provider and what is managed by an organization/user? 
ANS. With infrastructure as a service cloud model cloud service provider manages Virtualization, Servers, Storage, Networking and an organization/user manages Application, Data, Runtime, Middleware, O/S.
	

4. With Platform as a service cloud model what is managed by a cloud service provider and what is managed by an organization/user?
ANS. With infrastructure as a service cloud model cloud service provider manages Runtime, Middleware, O/S, virtualization, Servers, Storage, Networking and an organization/user manages Application, and Data.
	

5. With software as a service cloud model what does an organization/user manage and what is managed by a cloud service provider?
ANS. With software as a service cloud model a cloud service provider manages Applications, Data, Runtime, Middleware, O/S, Virtualization, Servers, Storage, Networking. Nothing is managed by the organization/user because all the resources being used are provided by the cloud service provider.
